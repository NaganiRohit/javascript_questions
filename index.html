JavaScript Interview Questions

Question 1: What is JavaScript, and how is it used in web development?
Answer (English):
JavaScript is a high-level, versatile programming language that enables dynamic interactivity on websites. It is primarily used for client-side development, allowing developers to create and control dynamic website content, such as interactive forms, animations, and real-time updates without reloading the page. It’s widely used alongside HTML and CSS to create full-fledged web applications.

Question 2: Explain the concept of "closures" in JavaScript.
Answer (English):
A closure in JavaScript is a function that remembers its lexical scope even when the function is executed outside of its original scope. This happens because functions in JavaScript have access to variables defined in their parent scope, even after the outer function has finished executing. Closures are often used to create private variables and functions.

Question 3: What is the difference between let, var, and const in JavaScript?
Answer (English):
- var is function-scoped, meaning it is accessible throughout the function in which it’s declared.
- let and const are block-scoped, meaning they are only accessible within the block where they are declared.
- let allows reassignment, while const does not.
- var can be hoisted, but it might lead to unexpected results, whereas let and const are hoisted differently and cannot be used before they are declared.

- let और const का स्कोप ब्लॉक के अनुसार होता है, इसका मतलब यह है कि यह केवल उस ब्लॉक में एक्सेस हो सकता है जहां इसे घोषित किया गया है।
- let को पुनः असाइन किया जा सकता है, जबकि const को नहीं।
- var को होइस्ट किया जा सकता है, लेकिन यह अनपेक्षित परिणामों का कारण बन सकता है, जबकि let और const अलग तरह से होइस्ट होते हैं और उन्हें उनके डिक्लेरेशन से पहले उपयोग नहीं किया जा सकता।
Question 4: What is the purpose of JavaScript's this keyword?
Answer (English):
The this keyword in JavaScript refers to the current context in which a function is being executed. Its value is determined by how the function is called. In global functions, this refers to the global object (e.g., window in browsers), while in methods, it refers to the object the method belongs to. Arrow functions do not bind this, so they inherit it from their containing scope.

JavaScript में this कीवर्ड का उपयोग उस वर्तमान संदर्भ का उल्लेख करने के लिए किया जाता है जिसमें एक फंक्शन निष्पादित हो रहा है। इसका मान इस पर निर्भर करता है कि फंक्शन को कैसे कॉल किया गया है। ग्लोबल फंक्शंस में, this ग्लोबल ऑब्जेक्ट (जैसे, ब्राउज़र में window) को संदर्भित करता है, जबकि किसी मेथड में, यह उस ऑब्जेक्ट को संदर्भित करता है जिससे वह मेथड संबंधित है। एरो फंक्शंस this को बाइंड नहीं करते हैं, इसलिए वे इसे उनके कंटेनिंग स्कोप से इनहेरिट करते हैं।
Question 5: What is the difference between synchronous and asynchronous JavaScript?
Answer (English):
Synchronous JavaScript executes code in a sequential order, meaning each operation waits for the previous one to complete before continuing. Asynchronous JavaScript, on the other hand, allows certain operations, like I/O tasks, to run in the background without blocking the execution of subsequent code. This is often managed using callbacks, promises, or async/await syntax to handle tasks like network requests, timers, and file operations.
Explanation (Hindi):

Question 6: What is the difference between "==" and "===" in JavaScript?
Answer (English):
In JavaScript, == is the equality operator that checks for equality after converting both operands to the same type, while === is the strict equality operator that checks for equality without any type conversion. This means === will only return true if both the value and the type are the same.

Question 7: What is an IIFE (Immediately Invoked Function Expression) in JavaScript?
Answer (English):
An IIFE (Immediately Invoked Function Expression) is a function in JavaScript that runs as soon as it is defined. It is typically used to create a private scope and avoid polluting the global scope. The function is wrapped in parentheses and followed by another set of parentheses to immediately execute it.
Explanation (Hindi):
IIFE (इमीडियेटली इनवोक्ड फंक्शन एक्सप्रेशन) एक ऐसा फंक्शन है जो जैसे ही परिभाषित होता है वैसे ही चलता है। इसका उपयोग आमतौर पर एक निजी स्कोप बनाने और ग्लोबल स्कोप को प्रदूषित करने से बचने के लिए किया जाता है। इस फंक्शन को परेथेसिस में लपेटा जाता है और इसके बाद इसे तुरंत निष्पादित करने के लिए एक और परेथेसिस दिया जाता है।
Question 8: Explain "hoisting" in JavaScript.
Answer (English):
Hoisting in JavaScript is a behavior where variable and function declarations are moved to the top of their containing scope during the compile phase. This means that variables declared with var and functions can be accessed before they are defined in the code. However, variables declared with let and const are hoisted differently and cannot be accessed before their declaration.
Explanation (Hindi):
JavaScript में होइस्टिंग एक व्यवहार है जिसमें वेरिएबल और फंक्शन डिक्लेरेशंस को उनके कंटेनिंग स्कोप के शीर्ष पर ले जाया जाता है। इसका मतलब है कि var से घोषित वेरिएबल और फंक्शंस को उनके कोड में परिभाषित होने से पहले एक्सेस किया जा सकता है। हालांकि, let और const से घोषित वेरिएबल्स को अलग तरह से होइस्ट किया जाता है और उन्हें उनके डिक्लेरेशन से पहले एक्सेस नहीं किया जा सकता।
Question 9: What is a promise in JavaScript, and how does it work?
Answer (English):
A promise in JavaScript is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. A promise can be in one of three states: pending, fulfilled, or rejected. It helps manage asynchronous tasks by providing then, catch, and finally methods to handle results once the operation completes.
Explanation (Hindi):

Question 10: What is the use of the "async" and "await" keywords in JavaScript?
Answer (English):
The async and await keywords in JavaScript are used to handle asynchronous operations in a more readable way. The async keyword is placed before a function to signify that it returns a promise, while await pauses the execution of an async function until the promise is resolved. This makes asynchronous code look more like synchronous code.

Question 11: What is event delegation in JavaScript?
Answer (English):
Event delegation is a technique in JavaScript where a single event listener is added to a parent element to handle events triggered by its child elements. Instead of adding individual event listeners to each child, the event is managed through the parent, improving efficiency and performance, especially in cases with dynamically added elements.

Question 12: Explain "callback functions" in JavaScript.
Answer (English):
A callback function is a function passed as an argument to another function, which is then invoked within the outer function to complete a routine or action. Callbacks are essential in asynchronous programming, as they allow code to execute once an operation (e.g., a network request) completes.

Answer (English):
The new keyword in JavaScript is used to create a new instance of an object from a constructor function. It sets the value of this within the constructor to the new object and returns the new object instance, allowing developers to create multiple objects with similar properties and methods.

Question 14: Explain the concept of "prototypal inheritance" in JavaScript.
Answer (English):
Prototypal inheritance is a JavaScript feature where objects can inherit properties and methods from other objects. Every object has a prototype, which serves as a template object. This allows one object to access the properties and methods of another object, enabling a form of inheritance that is central to JavaScript's object-oriented programming.

Question 15: What is the purpose of "strict mode" in JavaScript, and how is it enabled?
Answer (English):
Strict mode is a way to opt into a restricted variant of JavaScript, where certain errors that are typically silent are thrown as exceptions, and some JavaScript behaviors are prevented. It is enabled by adding "use strict"; at the top of a script or function. Strict mode helps improve code quality by preventing certain actions and catching common mistakes.

Question 16: What is the "spread" operator in JavaScript?
Answer (English):
The spread operator, represented by three dots (...), allows an iterable (like an array or object) to be expanded in places where multiple elements or arguments are expected. It is commonly used to clone arrays, concatenate arrays, or pass an array of arguments to a function.

Question 17: What is the "rest" parameter in JavaScript?
Answer (English):
The "rest" parameter, represented by three dots (...), allows a function to accept an indefinite number of arguments as an array. It is used in the function definition to gather all remaining arguments into a single array, making it helpful for handling variable numbers of parameters.

Question 18: What is a "generator" function in JavaScript?
Answer (English):
A generator function in JavaScript is defined using function* and can pause its execution at each yield expression and later resume from where it left off. Generators provide a way to handle asynchronous operations more effectively and are commonly used with for...of loops to iterate over custom iterable objects.
Explanation (Hindi):

Question 19: Explain the difference between "call", "apply", and "bind" methods in JavaScript.
Answer (English):

Question 44: What is "JSON" and how is it used in JavaScript?
Answer (English):
JSON (JavaScript Object Notation) is a lightweight data interchange format that is easy for humans to read and write and easy for machines to parse and generate. In JavaScript, you can use `JSON.stringify()` to convert a JavaScript object to a JSON string and `JSON.parse()` to convert a JSON string back into a JavaScript object.

Question 45: What is "event bubbling" in JavaScript?
Answer (English):
Event bubbling is a type of event propagation in the DOM where the event starts from the target element and bubbles up to its parent elements. This means that the event triggers on the target element first, and then on its ancestors in the DOM hierarchy.
Explanation (Hindi):
इवेंट बबलिंग DOM में इवेंट प्रसार का एक प्रकार है जहाँ इवेंट लक्ष्य तत्व से शुरू होता है और इसके माता-पिता तत्वों की ओर बढ़ता है। इसका मतलब है कि इवेंट सबसे पहले लक्ष्य तत्व पर ट्रिगर होता है, और फिर DOM पदानुक्रम में इसके पूर्वजों पर।
Question 46: What is "event capturing" in JavaScript?
Answer (English):
Event capturing is a phase of event propagation in the DOM where the event starts from the topmost parent element and descends down to the target element. This is the opposite of event bubbling.

इवेंट कैप्चरिंग DOM में इवेंट प्रसार का एक चरण है जहां इवेंट सबसे ऊपरी माता-पिता तत्व से शुरू होता है और लक्ष्य तत्व की ओर उतरता है। यह इवेंट बबलिंग का विपरीत है।
